shell 
- 리녹스 커널과 사용자를 연결해주는 인터페이스
- 커널 = 터미널
- 컴퓨터의 운영체제마다 있는 시스템에 접근하고 명령하여 프로그램을 만든다
- 사용자의 입력을 읽고, 해석하고, 프로그램을 실행시킴
- "shell 스크립트" : 일련의 명령행을 한 파일에 담아놓은 것
- 왜 파일에 넣을까? : 시간단축, 파일에 넣으면 한줄 씩 아니라 한번에 돌릴 수 있음, 컴퓨터가 꺼져있어도 돌아감
- 운영체제 : 리눅스 (기반,OS도 리눅스 중 하나)
- 리눅스 운영체제의 표준은 bash (bourne agaln shell)

문법
<프로그램> <내용> // 스페이스로 구분함, 스페이스까지 한번에 인식하고 싶으면 큰 따옴표 쓰기
echo : 셀 명령, 내장 명령, 프로그램
  - 화면에 출력하는 명령
  * 변수를 지정할 때에는 스페이스가 없어야 함 (shell에서는 스페이스를 주의해라)
  변수
  1. 내가 지정하는 변수
  2. 이미 지정되어 있는 변수
  - 변수를 내용에 넣을 수 있다.
   - 스페이스 주의할 것
   - ""는 사용해도 되고 스페이스 없으면 없어도 됨
   - 변수를 사용할 때에는 \$ 기호 사용한다
   - 내장된 변수도 존재한다. 이것들은 대문자로 지정되어 있다                           

  - 명령의 특징
   - 한 프로그램이 한 가지 일만 한다.
   - 명령을 잘 찾아야하고 출력 값에 알맞은 문법에 따라서 잘 조합해야 함
   - 명령이 너무 많아서 암기하지는 않고, 구글링으로 찾아서 잘 조합하는 것을 연습하자

  - 명령
   - 프로그램 자체
   - 단순 명령(simple command) : 프로그램 + 내용
   - 결합 명령 (combined command) : 프로그램 + 옵션 + 내용
    - 프로그램을 구체화하는 역할
    - 형태 : '-h'(옵션)
    "ls -l"
  - ls : list (나열, 프로그램)
  (if list를 한 문장씩 나눠서 하고 싶다면 list를 한 다음에 추가 기능을 하면 됨, -l)
  - change directory

  - 파이프 (명령을 잘 조합하는 방법)
   - |
   - 명령행 | 명령행
   - less (한 화면으로 깔끔하게 출력하고 싶을 때, 큌?을 해야 출력됨)
   - 이전 명령행의 출력값을 다음 명령행의 입력값으로 보내는 역할
   - 'echo $PATH | less'
   - 파이프라인 : 파이프라인으로 연결한 명령

 - wc, head, cut, grep, sort, uniq  

 - 입력
  - stdin : standard input/in. 표준입력

- 출력
 - stdout : standard output/out. 표준출력
 - 출력 스트림

* 파이프라인을 연결할 때 표준 입력으로 들어감
